# Manual OpenAI Submission Checklist

This checklist covers the manual steps that cannot be completed from the repository.

- [ ] Verify the OpenAI Platform organization or individual identity that will publish Convy.
- [ ] Confirm the Platform project uses global data residency.
- [ ] Confirm the submitting user has `api.apps.write`.
- [ ] Confirm the reviewing user has `api.apps.read`.
- [ ] Deploy the branch containing the final MCP widget, legal, landing, and docs updates.
- [ ] Create or verify `demo@convyapp.com`.
- [ ] Store the demo account password outside the repository.
- [ ] Seed the reviewer household, shopping list, task list, entries, and activity.
- [ ] Verify Firebase authorized domains include `auth.convyapp.com`.
- [ ] Connect `https://mcp.convyapp.com/mcp` in ChatGPT Developer Mode.
- [ ] Complete OAuth with the reviewer account.
- [ ] Run all positive reviewer prompts.
- [ ] Run all negative reviewer prompts.
- [ ] Capture screenshots listed in `submission-screenshot-checklist.md`.
- [ ] Open the OpenAI Platform Dashboard app submission flow.
- [ ] Create or update the Convy app draft.
- [ ] Fill app info from `openai-dashboard-submission-fields.md`.
- [ ] Add MCP server URL `https://mcp.convyapp.com/mcp`.
- [ ] Add OAuth notes and reviewer credentials.
- [ ] Upload screenshots.
- [ ] Add positive and negative test prompts.
- [ ] Confirm privacy, terms, and company URLs are public.
- [ ] Submit for review.
- [ ] Save the review Case ID outside the repository.
- [ ] Monitor dashboard status and review email.
- [ ] If approved, publish the approved version from the dashboard.
- [ ] If rejected, record feedback, update the repo, redeploy, and resubmit the same app draft/version path.
